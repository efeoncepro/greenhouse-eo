# Production risk register

| Riesgo | Gate preventivo | Respuesta |
| --- | --- | --- |
| Clones del colibrí | exactamente un cuerpo/cabeza/cola | rechazar; sólo alas repiten fases |
| Deriva entre formatos | todos parten del mismo anchor | comparar identidad y hook en contact sheet |
| Copy/logo incorrectos | composición determinista | no aceptar texto horneado por el modelo |
| Safe zones de plataforma | overlays medidos por formato | bloquear export que invade zonas reservadas |
| Look IA genérico | dirección de luz, grano, asimetría, detalle | iterar una variable o handoff humano |
| Sobreproducción | Lite diverge; Pro/GPT sólo tras curar | registrar costo por candidato aprobado |
| URL/claim sin destino real | usar sólo `efeoncepro.com`; sin CTA falso | media activation queda pendiente |
| Licencia/derechos | providers con commercial use documentado; assets oficiales propios | revisión legal si se reutiliza para cliente |
| Plataforma cambia specs | source check con fecha | reverificar al activar pauta |
| Reproducibilidad parcial | request IDs, prompts, hashes y scripts | no prometer seed determinista cuando API no lo expone |
| Deriva o fallo del master Omni | gate temporal sobre primer/medio/ultimo frame; clean plate y hashes preservados | fallback planificado a Seedance reference-to-video cuando la identidad espacial requiera mayor fidelidad; no se invoco porque ambos masters Omni pasaron el gate |
| Mezcla inconsistente entre aspectos | medir LUFS y peak sobre los seis releases, no solo presencia AAC | heroes normalizados; masters/bumpers quedan measurement-only y requieren mezcla/escucha humana antes de trafficking |
