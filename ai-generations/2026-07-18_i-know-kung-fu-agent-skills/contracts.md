# Contratos de infografía editorial — I Know Kung Fu

## Sistema visual

- Artículo: `«I Know Kung Fu»: el momento Matrix de los Agent Skills` (`post 250748`, privado).
- Tesis visual: el valor no está en descargar capacidades genéricas, sino en convertir criterio organizacional en método ejecutable y verificable.
- Skin: `efeonce_core`; canvas blanco deliberado; tinta navy; naranja sólo para decisión o activación.
- Shell: eyebrow, título autónomo, bajada breve, cuerpo semántico, nota, firma y sello `efeoncepro.com`.
- Tipografía: Poppins desde las fuentes versionadas en `src/assets/fonts/`. El source conserva texto editable; el delivery convierte cada glifo a paths para eliminar dependencia de fuentes en WordPress, navegador o sistema operativo.
- Fit tipográfico: labels críticos de círculos y cápsulas conservan un inset óptico medido; el build bloquea si superan el umbral de ocupación definido en `typography-fit-report.json`.
- Exclusiones: estética Matrix literal, lluvia de código, UI ficticia, cards de dashboard, glass, glow, gradientes, logos de plataformas y cifras no verificadas.

## KFU-V01 — Las tres capas de un agente útil

- Placement: cuerpo, después de `MCP le da manos al agente; los skills le dan método`.
- Estado: `production`.
- Tesis autónoma: un resultado gobernado aparece cuando razonamiento, acceso y método trabajan juntos.
- `explanatoryDelta`: muestra la convergencia simultánea de tres capas; la tabla adyacente sólo compara MCP y skills de forma binaria.
- Relación: tres capacidades complementarias convergen en un resultado utilizable y auditable.
- Qué no demuestra: que las tres capas sean productos independientes o que garanticen calidad por sí solas.
- Arquetipo: `layered-model` con convergencia triangular.
- Ruta: modelo → MCP → skill → resultado gobernado.
- Variante desktop: `1600×1000`; variante mobile: `1000×1500`.
- ALT: `Modelo, MCP y Agent Skill aportan razonamiento, acceso y método; las tres capas convergen en un resultado utilizable, verificable y auditable.`
- Caption: `La conexión habilita y el modelo razona; el skill aporta el criterio y las verificaciones del trabajo.`

## KFU-V02 — Del criterio tácito a una capacidad reusable

- Placement: cuerpo, después de `Qué conocimiento vale la pena convertir en un skill`.
- Estado: `production`.
- Tesis autónoma: el conocimiento propietario se vuelve capacidad cuando entra en un loop de observación, explicitación, ejecución y mejora.
- `explanatoryDelta`: hace visible el feedback que transforma experiencia dispersa en una capacidad mantenible; el artículo enumera criterios, pero no muestra el mecanismo de aprendizaje.
- Relación: ciclo de cinco movimientos alrededor del criterio propietario.
- Qué no demuestra: que todo conocimiento pueda automatizarse ni que el primer skill quede terminado.
- Arquetipo: `cycle`.
- Ruta: observar → extraer → empaquetar → ejecutar → revisar → observar.
- Variante desktop: `1600×1000`; variante mobile: `1000×1500`.
- ALT: `Cinco movimientos convierten conocimiento tácito en una capacidad reusable: observar decisiones, extraer criterios, empaquetar el skill, ejecutar con evidencia y revisar para actualizar.`
- Caption: `El skill no congela el conocimiento: lo convierte en un sistema que puede revisarse cuando cambia la operación.`

## KFU-V03 — ¿Este proceso merece convertirse en un skill?

- Placement: derivado social; candidato secundario para cuerpo si reemplaza la lista y no la duplica.
- Estado: `production`.
- Tesis autónoma: recurrencia, criterio, excepciones, verificación y ownership son señales de una buena candidata.
- `explanatoryDelta`: convierte una lista de condiciones en un recorrido de decisión autónomo que puede circular y utilizarse sin leer el artículo.
- Relación: camino de cinco gates, con salida explícita cuando falta mantenimiento o verificabilidad.
- Qué no demuestra: que las cinco señales tengan igual peso ni que constituyan una fórmula universal.
- Arquetipo: `path`.
- Ruta: cinco señales → decisión de prototipar y probar.
- Formato: `1080×1350`, crop-safe para thumbnail social.
- ALT: `Cinco señales ayudan a decidir si un proceso merece convertirse en Agent Skill: se repite, depende de criterio, contiene excepciones, produce un resultado verificable y tiene un dueño.`
- Caption: `Cuantas más señales reúna el proceso, mejor candidato es para un skill propietario.`
