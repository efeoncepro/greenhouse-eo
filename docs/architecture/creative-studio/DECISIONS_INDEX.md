| SPEC-005 | Accepted and implemented (TASK-1501) | Modality-discriminated run contract. `PrepareExperimentPayloadV1` gains `output?: OutputShapeV1`, a discriminated union by `modality` (image/video/audio) designed for all three from day one (image implemented end to end; video/audio declared + structurally validated, their adapter execution deferred to TASK-1504). Every output-shape param is validated FAIL-CLOSED in `prepare` (`validateOutputShape` inside `validatePreparePayload`) against the selected route's governed catalog constraints (TASK-1500, consumed in-process via a `RouteCatalogPort.getRoute` seam whose production impl is `getProducerRoute` — no re-dispatch) — strictly before `executeExperiment` reaches the spend fence. Unknown route, capability/route mismatch, modality incoherence, out-of-range/enum param, unsupported input mode, malformed shape, or a missing catalog all map to `invalid_request`, never silent coercion. Additive-optional (a shapeless caller is unchanged); the shape is threaded to the provider seam image-first (`CreativeProviderRequestV1` gains quality/aspectRatio/count), absorbing TASK-1495's aspect ratio for image. Selectors carry no bytes (references via `authorizedInputs`, edit base via `editFrom`); credit unit stays route × shape; coverage unchanged (`ui`/`mcp` policy-blocked). | [`EFEONCE_GLOBE_CREATIVE_PRODUCER_ARCHITECTURE_V1.md`](EFEONCE_GLOBE_CREATIVE_PRODUCER_ARCHITECTURE_V1.md) (§ Contratos reales del run contract) |
| SPEC-006 | Accepted and implemented (TASK-1502) | Previewable estimate reader. The credit estimate is extracted from inside the `execute` command to a governed, read-only reader `globe.lab.experiment.estimate` that returns `credits = f(route × output-shape)` BEFORE any spend — no experiment created, no credit reserved, no state transitioned. `LabRunnerPort.estimate` now takes a prospective `LabQuoteInputV1` (not a stored experiment); `execute` derives the quote from its stored experiment (`quoteInputFromStored`), so the preview and the command share the exact same estimate computation, never duplicated. The output shape is validated fail-closed against the route's catalog constraints reusing TASK-1501's validator (`invalid_request`); an over-cap estimate is NOT rejected — it returns `withinHardCap:false`, a budget signal (a preview tells the truth, it never spends). The curated `LabEstimatePreviewV1` exposes the credit number, the fidelity route and budget signals and OMITS provider/model slug, vendor cost and margin. Credit unit is route × output-shape, never the model. Read-only/idempotent; kill switch OFF ⇒ `policy_blocked`; coverage unchanged (`ui`/`mcp` policy-blocked). It is the advance slice of TASK-1469, which consumes the same estimate path. | [`EFEONCE_GLOBE_CREATIVE_PRODUCER_ARCHITECTURE_V1.md`](EFEONCE_GLOBE_CREATIVE_PRODUCER_ARCHITECTURE_V1.md) (§ Contratos reales del estimate) |
| ADR-004 | Accepted for the internal-only release; commercial frontend host deferred (TASK-1506) | Globe frontend hosting + front door. For the internal-only release the web/BFF/SSO shell stays on Cloud Run (native Node server; the Next.js target is superseded for this shell) — migrating to Vercel is rejected because it does not fix the in-memory/`maxScale=1` HA ceiling and splits the trust boundary for zero internal benefit. The COMMERCIAL client-facing frontend (TASK-1505+) is a SEPARATE surface with a DEFERRED host+framework decision (Vercel-hosted Next.js on a global edge is a live candidate), revisited when 1505's UI is built and before TASK-1480 Production — choosing Cloud Run now does not foreclose it. `globe.efeoncepro.com` is delivered by the successor task (TASK-1507) via a Global External ALB + serverless NEG (GA path, not a Preview direct domain mapping); `globe-api-internal` stays IAM-private with a `run.app` audience, never a browser domain. Three distinct gates — internal URL / scalability-HA (gated by durable stores, TASK-1465, hard-blocks `maxScale > 1`) / external Production (TASK-1480). The Cloud Run services enter Terraform in TASK-1507, closing the `invokerIamDisabled` drift. This ADR authorizes no infra/DNS/OAuth apply. | [`EFEONCE_GLOBE_FRONTEND_HOSTING_FRONT_DOOR_DECISION_V1.md`](EFEONCE_GLOBE_FRONTEND_HOSTING_FRONT_DOOR_DECISION_V1.md) |
| ADR-002 | Accepted and implemented — live-verified through the seam (TASK-1490); service rollout tracked in `Handoff.md` | Cross-model edit/refine in the Model Lab. One transport-neutral semantics, `editFrom = { experimentId }`, for every editable model instead of provider vocabulary (`previousInteractionId` deprecated and mutually exclusive with it — sending both is `invalid_request`, never resolved by precedence). NO dedicated `edit` command: an edit IS an experiment, on the same authority (`globe.lab.experiment.run`), spend fence, state machine and manifest. The caller declares the edit's own `capability`/`referenceRoute`/`hardCapCredits` — precisely what makes cross-model refinement possible (inheriting them from the parent would forbid it). The edit base never enters `authorizedInputs`, which stays pure caller declaration, and travels as `CreativeProviderRequestV1.editReference`, so an adapter tells "the thing being edited" apart from "additional references" instead of hoping it lands at index 0. A derivative carries the platform-certified `derived-internal` rights posture plus the parent's inherited constraints, rather than being laundered as `internal-owned`. And the chainability of a provider run reference is certified by the ADAPTER (`providerRunChainable`), never inferred by the domain, so a doomed chain is refused before the fence reserves. | [`EFEONCE_GLOBE_MODEL_LAB_V1.md`](EFEONCE_GLOBE_MODEL_LAB_V1.md) (§ Edit / refine cross-model) |
