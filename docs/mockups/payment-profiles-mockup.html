Perfiles de pago por beneficiario

Define cómo pagar a cada colaborador o accionista (banco local, Deel, Wise, Global66) con maker-checker y datos sensibles enmascarados por defecto. Una sola vez se decide la ruta; después Órdenes de pago la aplica automáticamente al programar pagos.
Cómo se conecta este módulo
  • Obligaciones (TASK-748) → genera "qué se debe pagar"
  • Perfiles de pago (este módulo) → resuelve "por qué rail pagar al beneficiario"
  • Órdenes de pago (TASK-750) → toma el perfil activo y arma la orden con provider + método ya resueltos
El resolver lee el perfil active por (espacio, beneficiario, moneda) y congela un snapshot en la orden. Si no hay perfil activo, la orden bloquea hasta que alguien lo cree.
