# Efeonce Operating Code — Hiring, Onboarding y Performance

> Canon: `docs/operations/EFEONCE_OPERATING_CODE_V1.md`. Esta guia explica como People/Talent lo aplica en procesos reales.

## Uso en contratacion

El Operating Code no reemplaza el scorecard tecnico. Agrega una capa de conductas job-related para roles de Efeonce: transparencia, educacion, memoria, impacto y sistema.

### Competencias culturales evaluables

| Competencia | Evidencia aceptable | Pregunta sugerida |
|---|---|---|
| Transparencia operativa | Explica riesgos, tradeoffs y estado sin esconder informacion | Cuentame de una vez que tuviste que mostrar un problema antes de tener la solucion. |
| Educacion / transferencia | Ensena a clientes o pares y deja criterio instalado | Como haces que un cliente/equipo entienda mejor una decision compleja? |
| Memoria operativa | Documenta aprendizajes, decisiones o assets reutilizables | Que dejaste reutilizable en tu ultimo proyecto importante? |
| Impacto de negocio | Conecta craft con outcome, no solo output | Como sabes que tu trabajo movio algo relevante? |
| Pensamiento de sistema | Reduce silos y entiende dependencias | Cuentame de una decision donde tu parte afectaba a otros equipos o sistemas. |

### Red flags

- Retiene conocimiento para volverse indispensable.
- Habla de cultura como "buena onda" y no como conducta.
- Reporta actividad sin outcome.
- Culpa al cliente por no entender.
- No puede explicar como deja aprendizaje reutilizable.

## Uso en onboarding

### Dia 0

- Presentar el Why canonico.
- Leer el Operating Code.
- Mostrar Greenhouse/Kortex/Verk como prueba del sistema.
- Explicar que documentar memoria es parte del trabajo, no administracion extra.

### Semana 1

- Revisar un caso real con Loop + ICO + Revenue Enabled.
- Identificar donde quedo la memoria del caso.
- Hacer un ejercicio: transformar un entregable en aprendizaje reutilizable.

### Dia 30

El manager revisa:

- Que entiende del Why.
- Que decision/documentacion dejo en su primer ciclo.
- Que parte del sistema uso correctamente.
- Donde necesita mas contexto para operar como Efeonce.

## Uso en performance

### Dimensiones recomendadas

| Dimension | Pregunta de evaluacion |
|---|---|
| Transparencia | Mostro estado, riesgos y decisiones con suficiente contexto? |
| Educacion | Elevo el criterio del cliente/equipo? |
| Memoria | Dejo documentacion, datos, templates, playbooks o aprendizajes reutilizables? |
| Impacto | Conecto su trabajo con negocio, eficiencia o calidad de decision? |
| Sistema | Mejoro el backbone operativo o redujo fragmentacion? |

### Calibracion

- Alto desempeno: resultado fuerte + memoria reusable + cliente/equipo mas capaz.
- Desempeno incompleto: buen output, pero sin aprendizaje acumulado.
- Riesgo cultural: resultados aparentes basados en opacidad, dependencia o heroismo recurrente.

## Ritual de proyecto

### Kickoff

- Que debe aprender el cliente?
- Que memoria debe quedar?
- Que dato conectara esto con negocio?

### Weekly

- Que riesgo hay que mostrar con contexto?
- Que aprendizaje debe registrarse?
- Que problema es sistemico?

### Cierre

- Que quedo reutilizable?
- Que decision puede tomar mejor el cliente ahora?
- Que parte del sistema mejora el proximo ciclo?

## Boundaries

- No usar "culture fit" como proxy de afinidad, personalidad, edad, origen, clase, genero, estilo comunicacional o background.
- Evaluar el Operating Code con evidencia conductual.
- Mantener scorecards estructurados y decision humana.
- Compensacion, contratos, compliance y payroll siguen en sus dominios especializados.
