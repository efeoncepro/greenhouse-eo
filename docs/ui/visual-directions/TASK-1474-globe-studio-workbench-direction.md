# TASK-1474 — Globe Professional Studio Workbench Visual Direction

## Benchmark and intent

Globe adopta una mesa de dirección creativa premium: candidate dominante, contexto operativo y evidencia
visible. La shell Orbital Threshold existente manda y sus patterns nacen dentro de Globe.

## Alternatives

1. **Editorial contact sheet:** visual, pero débil para approvals/lineage.
2. **Production control room:** robusto, pero demasiado técnico para el brief.
3. **Creative desk — selected:** canvas dominante, context rail adaptable y candidate dock.

## Selected direction

- Un momento visual dominante; estimate/rights/approval nunca compiten como cards equivalentes.
- Rail contextual para brief, references, route/model/version, credits del run y recovery.
- Dock para candidates/review/release; DAG fuera del onboarding.
- Provider/model/version propuestos versus ejecutados son transparentes; prompt/IP/cost/margin siguen privados.

## Portability and evidence

Globe usa HTML/CSS/TS server-rendered y construye su propia biblioteca incremental. `Creative Desk`,
`Context Rail` y `Candidate Dock` son patterns Globe con anatomy/states/a11y/responsive. Greenhouse gobierna
registry, lifecycle, QA y evidencia; no aporta recipes o components heredados. Baseline futuro:
`globe.professional-studio-workbench`. First-fold desktop/mobile se acepta antes del cableado exhaustivo.

Markers: `globe-studio-workbench`, `studio-canvas`, `studio-context`, `studio-candidate-dock`,
`studio-estimate`, `studio-review`, `studio-state-*`.

## Copy ledger and anti-patterns

Copy reusable cubre empty, estimate-required/expired, approval, running, candidate-ready, rights/budget/model
blocked, released/revoked y sanitized error. Evitar prompt form hero, DAG onboarding, card wallpaper, modelo
oculto, porcentaje inventado, créditos como dinero y action sin recovery.
