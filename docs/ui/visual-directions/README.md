# Visual Direction Contracts

Durable source of visual intent for Greenhouse UI. Every new `ui-standard` or
`ui-platform` task selects:

- `source-led` — approved external source copied/versioned or durably referenced;
- `repo-native-benchmark` — two or three directions compared and one selected.

Required content:

1. decision and rejected alternatives;
2. visual thesis and first-fold order;
3. desktop and mobile targets;
4. hierarchy/action model;
5. density/depth/type/color roles;
6. token mapping;
7. signature details;
8. anti-patterns;
9. acceptance signature.

Use `TEMPLATE.md`. A chat URL, inaccessible Claude artifact or “make it modern”
does not satisfy this contract.
