# 09 · GTM como Servicio (para clientes)

Efeonce es agencia: el GTM no es solo para su propio crecimiento — es un **deliverable de altísimo valor** que posiciona a Efeonce como **partner estratégico** (no ejecutor). Todo lo de `01`–`08` aplica; este módulo agrega la capa que cambia cuando el GTM es **para un cliente**.

## Qué cambia cuando es para un cliente
| Dimensión | Interno (Efeonce) | Cliente (servicio) |
|---|---|---|
| Doctrina | ASaaS/bow-tie de Efeonce | **la del cliente** — no impongas la de Efeonce |
| Confidencialidad | interna | **NDA + datos segregados** |
| Método | rigor suficiente | rigor **+ transparente y presentable** (pagan por confiar) |
| Alcance | flexible | **acordado por SOW** (anti scope creep) |
| Entregable | plan operable | **estrategia + plan pulido + presentación** a directorio |
| Ejecución | motores internos | el cliente ejecuta (o Efeonce ejecuta como servicio aparte) |

## Los engagements GTM que Efeonce puede vender
- **GTM audit / diagnóstico:** dónde está roto el GTM del cliente (funnel, motion, positioning, economía) — abre la puerta a más.
- **GTM strategy:** ICP/segmento + positioning + motion + canales + plan (todo `01`–`08`) como deliverable.
- **Positioning & messaging** (`02`) — muy demandado y de alto valor.
- **Launch strategy** (`06`) — lanzar un producto/mercado.
- **Demand strategy / funnel-bow-tie redesign** (`05`) — arreglar la máquina de demanda.
- **GTM retainer:** acompañar la ejecución y la cadencia (`08`) — el de mayor recurrencia.
- Cada uno se puede vender **como estrategia** (Efeonce diseña) o **estrategia + ejecución** (Efeonce corre los motores: contenido, paid, growth, CRM).

## Estándar del deliverable (para cliente)
- **GTM plan presentable a directorio** (no un doc interno) — es lo que el cliente lleva a su board.
- **Método transparente:** supuestos, fuentes (`as-of`), benchmarks, confianza y **limitaciones** — la transparencia ES el producto (el cliente escéptico revisa el método primero).
- **Recomendación accionable con owners + cadencia**, no un data dump estratégico.
- **Honestidad:** si el diagnóstico incomoda (su ICP está mal, su motion no calza, su economía no cierra), dilo con tacto pero dilo. Un GTM complaciente es un riesgo para el cliente y para Efeonce.

## Confidencialidad y conflicto de interés
- **NDA** + datos del cliente segregados de otros clientes y de lo interno de Efeonce (contratos → `legal-privacy-ip-operator` + `commercial-expert`).
- **No mezcles clientes competidores:** la estrategia GTM de un cliente no se reusa para su rival; gestiona el conflicto explícitamente.
- Si el GTM recolecta datos (research de mercado/ICP), cumple privacidad (`legal-privacy-ip-operator`).

## Billability y valor
- El GTM estratégico es **alto margen y alto valor percibido** — diferenciación de agencia, no commodity. Encájalo en el modelo comercial (proyecto o retainer) con `commercial-expert`; el pricing refleja rigor + impacto, no horas.
- Un **GTM audit** inicial suele **abrir la puerta** a un engagement mayor (el gap revelado justifica la estrategia + ejecución que lo cierra) — igual que el benchmark en `research-benchmark-operator` `11`.
- Es coherente con la propuesta Growth OS de Efeonce: vender el **sistema de crecimiento**, no tácticas sueltas.

## Checklist de salida (engagement de cliente)
- [ ] Alcance acordado en **SOW**; doctrina = la del **cliente**, no la de Efeonce.
- [ ] **NDA + confidencialidad + segregación**; sin conflicto entre clientes competidores.
- [ ] GTM plan **presentable a directorio**; método transparente (supuestos/fuentes/limitaciones).
- [ ] Recomendación accionable con **owners + cadencia**; honesto aunque incómodo.
- [ ] Definido si es **estrategia** o **estrategia + ejecución** (motores).
- [ ] Encajado en el modelo comercial (`commercial-expert`); privacidad cumplida.
